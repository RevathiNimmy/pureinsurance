# User Stories — New User Authority to Extract Client Data

**Feature**: New User Authority to Extract Client Data
**Spec ID**: SPEC-39906
**Source PBI**: ADO #39413
**Date**: 2026-06-01

---

## Personas

### System Administrator (SA)
- Manages user accounts and permissions via User Maintenance in the Admin Portal
- Responsible for granting/revoking access to sensitive operations
- Requires audit trail for compliance and governance

### Portal User — Authorised (PU-A)
- Has been granted the "Can Extract Client Data" authority by the System Administrator
- Accesses Client Details page in the Pure Portal
- Performs GDPR Subject Access Request data extractions when required

### Portal User — Unauthorised (PU-U)
- Has NOT been granted the "Can Extract Client Data" authority (default state)
- Accesses Client Details page in the Pure Portal
- Should not see or be able to trigger the data extraction functionality

---

## User Stories

### US-001: Add "Can Extract Client Data" Authority Checkbox

**As a** System Administrator
**I want to** see a new "Can Extract Client Data" checkbox on the User Maintenance Authorities tab in the Admin Portal
**So that** I can control which users are permitted to extract client data from the Portal

**Acceptance Criteria:**
- [ ] "Can Extract Client Data" checkbox appears on Tab 5 – Authorities, Policy sub-tab (Sub-Tab 3), Access section
- [ ] Checkbox follows the same pattern and positioning as "User can view Batch Process Status"
- [ ] Checkbox is unchecked by default for new users
- [ ] Existing users with no stored value are treated as unchecked (secure default)
- [ ] Checking the checkbox and saving persists the value to the user authority configuration table
- [ ] Unchecking the checkbox and saving persists the value to the user authority configuration table

**Priority**: High
**Estimate**: 5 story points
**Requirements**: FR-001 to FR-006

---

### US-002: Hide Extract Client Data Button for Unauthorised Users

**As a** Portal User without the "Can Extract Client Data" authority
**I want** the EXTRACT CLIENT DATA button to be hidden on the Client Details page
**So that** I cannot access functionality I am not authorised to use

**Acceptance Criteria:**
- [ ] EXTRACT CLIENT DATA button is completely hidden (not rendered) when authority is unchecked
- [ ] No alternative means to initiate client data extraction is available
- [ ] Authority is checked on page load of Client Details and cached for the session
- [ ] If server-side handler is reached without authority, request is rejected with message: "You do not have permission to extract client data. Contact your System Administrator."

**Priority**: High
**Estimate**: 5 story points
**Requirements**: FR-007, FR-010, FR-011, FR-012

---

### US-003: Show Extract Client Data Button for Authorised Users

**As a** Portal User with the "Can Extract Client Data" authority
**I want** the EXTRACT CLIENT DATA button to be visible and functional on the Client Details page
**So that** I can extract client data when required for GDPR Subject Access Requests

**Acceptance Criteria:**
- [ ] EXTRACT CLIENT DATA button is displayed in its standard position when authority is checked
- [ ] Clicking the button proceeds with the existing extraction workflow (password prompt, zip file generation, download) — no change to current behaviour
- [ ] Server-side handler processes the request normally for authorised users

**Priority**: High
**Estimate**: 3 story points
**Requirements**: FR-008, FR-009

---

### US-004: Audit Trail for Authority Changes

**As a** System Administrator
**I want** changes to the "Can Extract Client Data" authority to be recorded via system events
**So that** there is a traceable record of who granted or revoked this permission and when

**Acceptance Criteria:**
- [ ] System event is generated when the authority is changed (checked or unchecked)
- [ ] Event captures: administrator who made the change, affected user, old value, new value, timestamp
- [ ] Audit follows the same system event pattern used for other User Maintenance configuration changes

**Priority**: Medium
**Estimate**: 3 story points
**Requirements**: FR-013, FR-014
