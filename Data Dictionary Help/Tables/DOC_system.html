DOC_system

Holds the permission levels and details of the next available page.

Logical database: DocuMaster
Component team: DocuMaster
Software module: DocuMaster

Definition

Name Data Type Defaults/Rules Constraints Description
system_id integer not null PK Primary key for the table.
doc_date tinyint not null   Default number of days to subtract from todays date for new documents added to the system.
This is set to 2 by default. The reason for this is that they may receive the post at the office, but not get round to processing it (adding it to DME or what have you) for a few days.
expiry_date smallint not null   Default expiry date for documents.
admin_level tinyint not null   The level at which users become an administrator for the DME system.
next_page char(15) not null   File path to be used when the next page is created.
update_history char(1) null   Indicates whether the doc_history table should be updated after given actions. This is for UNIX integration.
File_Delete_Level tinyint null   The user level required to perform a file delete action.
File_Move_Level tinyint null   The user level required to perform a file move action.
File_Copy_Level tinyint null   The user level required to perform a file copy action.
Folder_Delete_Level tinyint null   The user level required to perform a folder delete action.
Folder_Move_Level tinyint null   The user level required to perform a folder move action.
Folder_Copy_Level tinyint null   The user level required to perform a folder copy action.
