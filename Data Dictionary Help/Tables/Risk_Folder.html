Risk_Folder

Performs the same function for risks as insurance folder does for policies, i.e. it links versions of risks.

Logical database: BackOffice
Component team: Pure For Underwriting
Software module: Risk

Definition

Name Data Type Defaults/Rules Constraints Description
risk_folder_cnt integer identity not null PK Unique ID - automatically generated by database
risk_folder_id integer not null AK risk folder id - unique only within each source id (generate using spu_get_unique_number)
source_id integer not null AK, FK, I ð Source.source_id source id, which branch is this - same as on insurance file
risk_folder_type_id integer null FK, I ð Risk_Folder_Type.risk_folder_type_id which risk folder type is this - always 1
code varchar(40) not null AK Code must be unique. Usually generated from 'C' + source_id + ' ' + risk_folder_id.
description varchar(255) null   Description Field
insurance_folder_cnt integer null FK, I ð Insurance_Folder.insurance_folder_cnt which insurance folder does this link to
