DOC_volume

Holds the volume details.

Logical database: DocuMaster
Component team: DocuMaster
Software module: DocuMaster

Definition

Name Data Type Defaults/Rules Constraints Description
volume_id integer not null PK Primary key for the table.
volume_name varchar(30) null   Name fo the volume. Generally starts with "DMS HD1" and then is incremented by 1 if the DME folders are spread out on more than one physical disk.
directory varchar(15) null   Directory for the data. This is beneath the path described by the doc_device table.
device_id tinyint not null   id for the device. This is usually just 1 (as theres only ever 1 device on 99.9% of systems)
