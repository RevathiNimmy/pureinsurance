DOC_folder

Holds the folder details and hierarchy. Used to build the folder structure seen in the Doument Manager.

Logical database: DocuMaster
Component team: DocuMaster
Software module: DocuMaster

Definition

Name Data Type Defaults/Rules Constraints Description
folder_num integer identity not null PK Unique ID - automatically generated by database
folder_name varchar(285) not null FK ð Party.shortname The name for the folder. This is the description that appears within the DME Manager.
NOTE: In some cases, when a level 1 folder is linked to a party, the ex_code column will not be populated with the party_cnt value. In this situation, use the folder name to link to Party.shortname.
parent_num integer not null FK(L), I ð DOC_folder.folder_num Points to the parent folder in the hierachy. A zero value indicates the top level.
ex_code varchar(20) null FK(L), I ð Source.source_id or Party.party_cnt or Insurance_File.insurance_ref or Insurance_Folder.insurance_folder_cnt or Claim.Claim_id The meaning depends on the folder level. However, the data may be a mixture of different conventions depending on what version of DME was used to write the data.

LEVEL 0:
Source (Source.source_id)

LEVEL 1:
IF number formatted as 'L000000000' THEN Swift Client (S_CLIENT.CLIENT_NUM)
ELSE Party (Party.party_cnt)

LEVEL 2:
IF 'GENERAL' THEN general party folder
ELSE IF number formatted as 'C00000000' THEN Claim (Claim.claim_id)
ELSE IF number formatted as 'F000000000' THEN Swift Policy (S_POLMAI.POL_NUM)
ELSE IF number formatted as '0' THEN Insurance Folder (Insurance_Folder.insurance_folder_cnt)
ELSE Insurance File (Insurance_File.insurance_ref)

LEVEL 3 (underwriting 1.9.x only):
IF number formatted as 'C00000000' THEN Claim (Claim.claim_id)
ELSE Risk (Risk.risk_cnt)

LEVEL 4 (underwriting 1.9.x only):
Loss Schedule (????)

If the DMESync application has not yet been run, then Swift Client looks like Party.
There are a lot of other conventions which are not documented here.
folder_level tinyint null   DMEFolderLevel The nesting level of the folder. Zero indicates the top level, and each lower level increments by one.
access_level tinyint null   The minimum access level required to access the folder.
password varchar(12) null   The password required to access the folder.
create_date datetime not null   The date the folder was created.
