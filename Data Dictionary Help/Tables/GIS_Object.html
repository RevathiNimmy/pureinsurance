GIS_Object

GIS objects (user-defined tables)

Logical database: GIS
Component team: Pure Architecture
Software module: GIS

Definition

Name Data Type Defaults/Rules Constraints Description
gis_object_id integer not null PK gis object
gis_data_model_id integer not null FK, I ð GIS_Data_Model.gis_data_model_id Parent GIS data model
object_name varchar(70) not null   Object name as displayed on screen
There are typically two default objects called <dmc>_policy_binder and <dmc>_output.
For user created objects the object name must not be preceded by <dmc>.
table_name varchar(70) not null   Table name as it appears in the database
There are typically two default objects called <dmc>_policy_binder and <dmc>_output.
For user created the table name is always <dmc>_object name.
max_instances integer null   This is a red hearing and is used by document production to indicate repeating child objects. A number greater than 2 causes SPs to be generated to cater for the retrival of multiple children.
is_quote_object tinyint not null   Boolean is quote object
parent_object_id integer null FK, I ð GIS_Object.gis_object_id parent object
polaris_object_id integer null   polaris object
is_selectable_for_screen tinyint null   Boolean is selectable for screen
Should be set to 1 if the object is to appear in the data model in the screen editor.
Should be set to 0 for system objects like <dmc>_policy_binder and <dmc>_output.
is_non_gis tinyint null   GISSpecialType as defined in GISDataModelType.bas
edit_flags tinyint not null   GISEditFlags This functionality is only available in 1.9
